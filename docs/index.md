## The Core Problem This Solves

### The "$2000 Mistake"

You have a great idea. You spend days prompting AI. You burn through $2000 in API tokens. You get... a half-built application that doesn't work.

**Why?**
- No MVP scoping - tried to build everything at once
- No quality gates - no way to know if phase 1 works before starting phase 2
- No context management - AI "forgot" early decisions
- No budget planning - didn't know cost until it was too late

### The "$200 Success"

**Better approach:**
1. Brainstorm with AI to identify **realistic MVP** (2-4 weeks of work)
2. Document **full vision** in README (v0.1 â†’ v1.0 â†’ Production)
3. Create **phased roadmap** (MVP â†’ Version 1 â†’ Production â†’ User Testing)
4. Build **MVP first** (~$200 in tokens)
5. **Validate concept** works before continuing
6. If successful, proceed to next phase

**Result:** You either prove the concept works for $200, or you learned it doesn't work before wasting $2000.

## The Methodology in Brief

### Phase 0: Pre-Development (The Game-Changer)
1. **Brainstorming Session** with extended thinking enabled
2. AI helps identify: "Your vision is lofty, let's build an MVP first"
3. Create **project DNA**: roadmap, templates, rules, learning log
4. Document **full vision** but only plan **MVP phases**
5. **Estimate costs** and set realistic expectations

### Phase 1-N: MVP Development
6. Execute systematically with **Cline workflow**
7. New chat per task (prevent context pollution)
8. Confidence scoring on every subtask (8/10 minimum)
9. Phase audits between major milestones
10. **MVP complete** - validate it works!

### Post-MVP: Scale When Proven
11. If MVP proves concept, proceed to v1.0 phases
12. If not, you learned for $200 instead of $2000
13. Repeat methodology for each version increment
